# Angular to Database Client Side Project. 
  
We are going to work in teams of two. Setup a repository on GitHub. You can choose to do this under your own personal account or under an organization that you create. The repository can be public or private, but either way, you should add me [@ogryzek](https://github.com/ogryzek) as a collaborator.  
  
Additionally, set up a project with tickets detailing the work to be done, and also make sure that I am a member of the project.  
  
We are going to use [Firebase](https://firebase.google.com/) for the database. Here's a [tutorial](https://firebase.google.com/codelabs/firebase-web#0). 
  
## Rules

The project details are described in the [Project Description PDF](https://learn.greystonecollege.com/pluginfile.php/100892/mod_resource/content/2/Angular%20to%20Database%20Client%20Side%20Project.pdf).  
  
Each day (Tue, Wed, Thu), should record a team daily standup. This can be a very short (30 seconds, to a minute roughly) video where you and your partner touch base on what you did yesterday, what you're doing today, and if there are any impediments.  
  
**IMPORTANT**: The daily standup must be posted to Teams by 6:00 PM. Daily attendence depends on this, as well as on your Github activity (commits, pull requests, comments, etc.).  
  
## Important Dates.

  - April 10th (12:00pm - noon): Final Submission - you should upload a zipfile with your final code for the "Submit Project" assignment, and a (up to 5 minute) video with a code walkthrough explaining the parts of the project that you worked on for the Project Presentation.
